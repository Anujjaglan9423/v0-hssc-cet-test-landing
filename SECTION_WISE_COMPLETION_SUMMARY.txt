═══════════════════════════════════════════════════════════════════════════════
  CSV FORMAT REQUIRED
═══════════════════════════════════════════════════════════════════════════════

Required Columns:
  question_text        - The question text
  option_a            - Option A text
  option_b            - Option B text
  option_c            - Option C text
  option_d            - Option D text
  correct_answer      - A, B, C, or D (case-insensitive)

Optional Columns:
  explanation         - Why this answer is correct
  exam_source         - Exam year or source

Example:
  question_text,option_a,option_b,option_c,option_d,correct_answer,explanation
  "What is 2+2?","2","3","4","5","c","2+2 equals 4"
  "Capital of India?","Mumbai","Delhi","Bangalore","Chennai","b","New Delhi is capital"

═══════════════════════════════════════════════════════════════════════════════
  DATABASE SCHEMA (NO MIGRATIONS NEEDED!)
═══════════════════════════════════════════════════════════════════════════════

Uses Existing Tables:
  • tests table - Test metadata
  • questions table - Individual questions

Data Storage:
  • Section metadata stored as JSON in tests.description
  • Section markers in questions.exam_source field
  • Questions linked to test via test_id
  • Question order maintained with question_order field

No new columns or tables required!

═══════════════════════════════════════════════════════════════════════════════
  HOW TO USE
═══════════════════════════════════════════════════════════════════════════════

FOR ADMINS:
  1. Go to Admin > Tests page
  2. Click "Add Section-wise Test" button
  3. Fill in test title and difficulty
  4. Define sections and names
  5. Upload CSV for each section
  6. Review questions in preview step
  7. Edit/delete any wrong questions
  8. Click "Create Test"
  9. Test appears in "Section-wise" tab

FOR DEVELOPERS:
  1. Import component: 
     import { SectionWiseCSVUploadModal } from "@/components/admin/section-wise-csv-upload-modal"
  
  2. Use in component:
     <SectionWiseCSVUploadModal
       open={showModal}
       onOpenChange={setShowModal}
       onTestCreated={() => loadTests()}
     />
  
  3. Or call server actions directly:
     await createSectionWiseTest({...})
     await uploadSectionCSV(testId, sectionName, questions)
     await getSectionWiseTests()
     await deleteSectionWiseTest(testId)
